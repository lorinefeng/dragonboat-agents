# DragonBoat Vision

DragonBoat is a local-first crew layer for cross-platform coding agents.

Its long-term purpose is to help AI-native developers become more capable solo builders by turning isolated coding-agent sessions into a coordinated, observable, and reviewable crew.

## The Belief

A flagship reasoning model should not have to do every piece of work alone.

The strongest agent should act like the steerer and drummer of a dragon boat: understanding the user's real goal, setting direction, breaking work into rhythm, handling difficult reasoning, and making the final call.

Lower-cost or more specialized worker agents should act like rowers: receiving enough context, working inside clear boundaries, reporting blockers, producing changes, running checks, and returning evidence.

## The Problem

Coding agents are already strong, but their sessions are usually isolated.

This creates real friction:

- context is copied by hand
- workers do not always know their boundaries
- progress is hard to monitor
- peer-to-peer handoffs are improvised
- final results are hard to verify
- users are nudged into one vendor's ecosystem

DragonBoat exists to make delegation less lossy.

## The Opportunity

The next generation of AI-native software work needs a coordination scaffold that belongs to the user, not to a single model provider, IDE, or cloud platform.

DragonBoat should let users decide:

- which agent is the steerer
- which agents are rowers
- which tasks deserve premium reasoning
- which tasks can be delegated to cheaper workers
- what evidence is required before work is accepted

## The Identity

The name DragonBoat is intentional.

A dragon boat is fast because the crew moves together. The point is not one heroic rower. The point is rhythm, trust, direction, and shared effort.

That cultural identity should stay visible, but the engineering product must remain serious, precise, and useful.

## The Long-Term Vision

DragonBoat should become an open-source coordination scaffold for AI-native builders who want to operate like a small software company before they hire one.

Not a chatbot.

Not an agent wrapper.

A local crew layer for real engineering work.
